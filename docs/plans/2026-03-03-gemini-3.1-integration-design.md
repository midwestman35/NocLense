# Gemini 3.1 Integration Design

**Date:** 2026-03-03
**Approach:** B ÔÇö Refresh + clean descriptions
**Status:** Approved

---

## Context

NocLense currently ships five Gemini models. Two are actively harmful to leave in place:

- `gemini-3-pro-preview` ÔÇö deprecated by Google, shuts down **March 9, 2026**
- `gemini-2.0-flash`, `gemini-1.5-pro` ÔÇö legacy, unavailable to new API users

This design integrates `gemini-3.1-flash-lite-preview` as the new default and adds `gemini-3.1-pro-preview` as the quality tier, producing a clean three-model Gemini list aligned with Google's current offering.

---

## Architecture

No new files. No provider, UI, or Electron changes. Three files only:

| File | Change type |
|---|---|
| `src/types/ai.ts` | Model registry ÔÇö add 2, remove 3, update default |
| `src/services/llmService.ts` | Update default model ID in 3 places + error copy |
| `src/contexts/AIContext.tsx` | Migration path for deprecated/removed model selections |

All changes follow existing patterns. JSDoc and "Why" comments are maintained per `.cursorrules` mandatory comment rules for AI files.

## Migration Path (`src/contexts/AIContext.tsx`)

Extends the existing `loadModel()` migration block (which already handles `gemini-1.5-flash` Ôćĺ `gemini-3-flash-preview`). New cases:

| Saved value | Migrates to | Reason |
|---|---|---|
| `gemini-3-pro-preview` | `gemini-3.1-pro-preview` | Deprecated, nearest equivalent |
| `gemini-2.0-flash` | `gemini-3.1-flash-lite-preview` | Removed legacy, nearest equivalent |
| `gemini-1.5-pro` | `gemini-3.1-flash-lite-preview` | Removed legacy, nearest equivalent |

Migration is silent ÔÇö new value is persisted so the user is not prompted again. Same pattern as existing migrations in the file.
