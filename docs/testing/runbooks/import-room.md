## Preconditions

## Steps

### 1. First paint

**Action:** Enter Import Room from Dashboard.

**Pass criteria:**
- Centered intake card renders with three tiles: Upload, Paste, Zendesk.
- Source chips row below tiles lists the 4 source types (APEX, Datadog, AWS, Unknown) per `SOURCE_OPTIONS` in `importRoomShared.ts`.
- Phase dots show Import active.
- No console errors.

### 2. Small file import (local parse path)

**Action:** Drag `small.log` onto the dropzone (or click Upload and select via native dialog).

**Pass criteria:**
- Dropzone shows active state during drag-over (border color / shadow change).
- On drop, `ImportProgress` renders a progress indicator.
- Parse completes in <3s on a modern machine; no IndexedDB mode switch (file below `FILE_STREAM_THRESHOLD_BYTES` = 50MB).
- `ImportSummary` renders showing: filename, size, entry count, source label, time range of first/last entry.
- `useIndexedDBMode` stays `false` (confirm via React DevTools or by verifying `LogContext.indexedDBLogs.length === 0`).

### 3. Large file import (Rust streaming path)

**Action:** Drag `large.log` (>100MB) onto the dropzone.

**Pass criteria:**
- File size detected as ≥50MB; streaming path engages.
- Rust-side `file_stream` emits `file-chunk` events; renderer ack backpressure visible as steady progress (no sudden jumps).
- `useIndexedDBMode` flips to `true`; `LogContext.useIndexedDBMode` reads true after import completes.
- Progress bar advances smoothly (no UI freeze); main thread responsive (can click other UI elements mid-parse).
- Final `ImportSummary` renders with correct entry count; IndexedDB `logs` store contains all entries (verify via IndexedDB inspector: `NocLenseDB > logs > total === entry count`).
- No memory spike >500MB in the Tauri process (check Task Manager / Activity Monitor).

### 4. Malformed file error path

**Action:** Drag `malformed.log` onto the dropzone.

**Pass criteria:**
- Parse fails with a user-readable error message in the Import Room UI (not a toast, not a console-only error).
- Error message names the problem (e.g., "Unsupported file encoding" or "File appears truncated at line 47").
- Partial state is cleared — no orphan entries in IndexedDB; `useIndexedDBMode` stays at whatever it was before.
- User can immediately retry with a different file without reloading the room.

### 5. Cancel mid-import

**Action:** Start importing `large.log`. While `ImportProgress` shows <50% complete, click Cancel (or hit Esc if wired).

**Pass criteria:**
- Import halts within 1s.
- Rust streaming command terminates cleanly (no orphan process visible in Task Manager).
- `useIndexedDBMode` resets to `false`.
- Any partially-written IndexedDB rows are purged (check `logs` store; count should match pre-import state).
- UI returns to the intake card state; user can start a new import.

### 6. Paste path

**Action:** Click the Paste tile. Paste contents of `paste-buffer.txt` into the input.

**Pass criteria:**
- Textarea or similar input accepts the paste.
- Submit button triggers parse via the same local parser used in Step 2.
- `ImportSummary` renders with source label matching user selection (or "Pasted" default).

### 7. Zendesk intake path (if wired)

**Action:** Click the Zendesk tile. Enter a known ticket ID. Submit.

**Pass criteria:**
- If Zendesk credentials are saved in `aiSettings` (per Setup Room runbook): ticket fetches, attachments list renders, user can select which attachments to import.
- If credentials not saved: clear error message pointing user to Setup Room.
- Imported attachments appear as separate batches with Zendesk source label and ticket ID in their batch metadata.

### 8. Source chip labeling

**Action:** Click a source chip (e.g., APEX) before importing. Import a file.

**Pass criteria:**
- Imported batch's `sourceType` matches the clicked chip.
- Batch appears in `importedDatasets` with correct `sourceLabel`.
- `LogContext.logs` (or IndexedDB rows) tagged with the source.

### 9. Multi-file import

**Action:** With one dataset already imported, drag a second file.

**Pass criteria:**
- Second import starts as a new batch (doesn't replace first).
- `LogContext` accumulates entries from both; total count matches sum.
- `importedDatasets` list in Dashboard / Submit Room shows both.

### 10. Phase-dot transition

**Action:** Complete any import successfully.

**Pass criteria:**
- Phase dot Import transitions to `completed` state; phase dot Setup transitions to `active`.
- User can auto-advance to Setup Room, or click the Setup dot (forward nav allowed per post-07C.2 polish decision).
- Returning to Import Room later shows the completed state + the imported batches persistently listed.

### 11. IndexedDB mode toggle (manual)

**Action:** If a manual toggle for `useIndexedDBMode` is exposed in the UI (verify against current chrome), flip it after importing a small file.

**Pass criteria:**
- Toggle flips `LogContext.useIndexedDBMode` to `true`.
- Previously-imported local-memory logs are persisted into IndexedDB transparently.
- `LogViewer` continues to render the same entries — no visible change to the user.

## Known failure modes

| Symptom | Root cause | Fix / watch |
|---|---|---|
| Drop event does nothing | `ImportDropzone` missing `onDrop` preventDefault or wrong DOM target | Grep `ImportDropzone.tsx` for `e.preventDefault()` |
| Large file causes browser OOM despite streaming | Rust backpressure broken — renderer not sending `file-chunk-ack`; `AckWaiter` Condvar hangs forever | Check `src/services/fileStream.ts` ack emit; confirm `file-chunk-ack` event reaches Rust |
| IndexedDB upgrade "blocked" on open | Another NocLense instance has the DB open | Close other instances; `indexedDB.ts:66` throws the user-facing error |
| Parse completes but entries missing | `parser.ts` regex fell out of date vs. new log format | Check source-specific parser branch (APEX / Datadog / AWS) |
| Progress bar stuck at 100% but no summary | `onComplete` callback never fires — check streaming `isLast` chunk handling | `file_stream.rs` / `fileStream.ts` |
